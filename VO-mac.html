How HTML elements are supported by screen readers

Typical support patterns of HTML elements by screen readers:

  • Identification of an element by role as the user moves through the content.
  • Announcement of the text content of an element.
  • Announcement of the start and end of an element.
  • Change in voice as the content of an element is announced.
  • Announcement of an elements accessible name and/or description
  • Announcement of states and properties.
  • Emission of a beep or other sound when an element with a particular state or property receives virtua focus.
  • Instructions on how to operate interactive elements such as form controls.
  • Navigation of elements by keyboard and “quick access” lists of a particular elements, list items are linked to each instance of an element on the page.

Note: The combination of patterns supported varies from element to element and support for a particular element varies between screen reader software.

Element Test Case Represents Aural UI Interaction Supported Notes
<a> with href a with href Hyperlink Announces link text + "link" VO + right/left arrow to navigate, VO + space to activate VoiceOver recognizes visited/unvisited state of links.
<a> without href a without href Anchor Reads content as plain text No special interaction Treated as regular text with no link functionality.
<abbr> abbr Abbreviation Reads element content; announces title as "group" if present before element content No special interaction Expands abbreviation if title attribute is present, announces as "group".
<address> address Contact information Reads content as plain text No special interaction No specific semantics beyond reading the text.
<article> article Self-contained composition Announces as "article" VO + Right Arrow to navigate Recognizes start/end of articles.
<aside> aside Tangentially related content Announces as "complementary" VO + Right Arrow to navigate Recognized as a complementary region.
<audio> audio Audio player Announces as "Audio Controls, group" if the controls attribute is present, and as label text + "audio" if an aria-label is present Focusable via VO + space to play/pause Buttons in the audio player are accessible.
<b> Bold Bold text Reads content as plain text No special interaction No specific emphasis conveyed.
<bdi> bdi Bi-directional isolation Reads element content No special interaction No semantics conveyed.
<bdo> bdo Bi-directional override Reads element content No special interaction No semantics conveyed.
<blockquote> Block quote Quoted section Reads content as plain text Navigable via VO commands No semantics conveyed.
<body> body [no test] Document body Reads content No special interaction No semantics conveyed.
<button> Button Button control Announces as label + "button" VO + space to activate Recognizes the button label and, if provided, state.
<canvas> canvas Scriptable bitmap canvas Announces as "image" if aria-label is present or as text if fallback content is provided. Otherwise, canvas is ignored Focusable via VO navigation Accessible fallback content is read if provided.
<caption> Table caption Table caption Announces caption when table is navigated to VO + Shift + Down Arrow to enter table, VO + Arrow Keys to navigate between table cells Acts as the table's title.
<cite> cite Reference to a creative work Reads element content No special interaction No special semantics conveyed.
<code> code Code snippet Reads element content No special interaction No specific code indication given.
<col> col Table column No aural indication No special interaction No semantics expected.
<colgroup> colgroup Group of columns in a table No aural indication No special interaction No semantics expected.
<data> data Machine-readable equivalent Reads element content No special interaction No semantics conveyed.
<datalist> datalist Combo box suggestions Announces as "list box pop-up" on Safari or "list box pop-up, combobox" on Chrome VO + space to open + Up/Down Arrows to navigate Provides suggestion list as the user types.
<dd> dd Description definition Reads element content Part of list navigation Included as part of the definition list.
<del> del Deleted text Announces "deletion" before deleted text when included as part of a tag with plain text, otherwise no aural indication No special interaction Uses a different tone to indicate deletion. Support is inconsistent across browsers and "deletion" is only ever announced when part of a tag with plain text.
<details> details Disclosure widget Announces summary content as disclosure label + "summary" in Safari or "group" on additional browsers VO + space to expand/collapse Recognizes expanded/collapsed state.
<dfn> dfn Defining instance of a term Reads element content No special interaction No specific semantics conveyed.
<dialog> dialog Dialog box Announces "dialog" on Chrome, announced as "group" on additional browsers Focus moved to dialog when opened Recognizes when dialog opens and traps focus within.
<div> div Generic flow container Reads element content No special interaction No special semantics unless ARIA roles are provided.
<dl> dl Definition list Announces as "definition list with X items" VO commands to navigate Recognizes list structure.
<dt> dt Definition term Reads element content, suffixed with "term" if an aria-label is present Part of definition list navigation Properly identifies as part of the definition list.
<em> em Emphasis Reads with emphasis No special interaction Uses different intonation for emphasis.
<embed> embed Embedded content Announces as "frame" Focusable via VO navigation Varies depending on the embedded object type.
<fieldset> fieldset Group of form controls Announces as "group" Navigates using VO commands Recognizes fieldset and legend content.
<figcaption> FigureCaption Figure caption Reads caption text No special interaction Announces when associated with a figure.
<figure> figure Figure with optional caption Announces "figure" Navigable via VO commands Recognizes start and end of the figure.
<footer> footer Footer for page or section Announces as "content information"
  • VO + right/left arrow to navigate, focusable with VO
  • VO + U to open rotor, Right/Left arrow to navigate to landmarks menu to see list of landmarks including the footer element
 Identified as a landmark region.
<form> form User-submittable form Announces as "form" if aria-label is present, otherwise not announced VO + Right Arrow Recognizes labeled forms and their controls.
<h1> to <h6> headings Section headings Announces "heading level X" VO + Right Arrow to navigate Properly identifies heading level and text.
<header> header Header for page or section Announces as "banner" VO + Right Arrow Identified as a landmark region.
<hr> hr Thematic break Announces as "horizontal separator" in Safari and "horizontal splitter" in Chrome Navigable via VO commands Properly indicates a thematic break.
<html> html [no test] Root element No special aural indication No special interaction No semantics expected.
<iframe> iframe Inline frame Announces as "frame" + the frame number or title if present Focusable via VO navigation Recognizes frame title when present.
<img> img Image Announces as "image" + alt text if present VO + Right Arrow Reads alt attribute content or "image" if none provided.
<input type="button"> input [button] Button control Announces label, if present + "button" VO + space to activate Recognizes button type and label.
<input type="checkbox"> input [checkbox] Checkbox control Announces as "unticked/ticked, tickbox" VO + space to check/uncheck Identifies the label and state.
<input type="color"> input [color] Colour selection control Announces as currently selected colour + "color well" VO + space to open colour picker Allows selection of colours from the palette.
<input type="date"> input [date] Date selection control Announces "date area" VO + Shift + Down Arrow to edit date, + Right/Left Arrows to navigate In Chrome, announces "edit text" when input is focused
<input type="datetime-local"> input [datetime-local] Date selection control Announces "date and time area" VO + Shift + Down Arrow to edit date and time, + Right/Left Arrows to navigate Announces "edit text" when input is focused.
<input type="email"> input [email] Email address entry Announces "edit text, email field" Focusable via VO navigation Announces "edit text" when input is focused.
<input type="file"> input [file] File selection/upload control Announces "file upload button" VO + space to open file selection control Allows file browsing and selection.
<input type="hidden"> input [hidden] [no test] Non-visible input No aural indication Not interactive Used for form data; ignored by VoiceOver.
<input type="image"> input [image] Submit button with an image Announces as "button" + alt text if present VO + space to activate Acts like a regular button.
<input type="month"> input [month] Month and year input Announces "edit month and year field" Focusable via VO navigation Announces "edit text" when input is focused.
<input type="number"> input [number] Numeric entry Announces as "incremental edit text number field" Focusable via VO navigation, Up/Down Arrow keys to increase/decrease value Editable text field when focused with VO or the incremental stepper can be interacted with by use of Up/Down arrow keys.
<input type="password"> input [password] Password entry field Announces as "secure edit text" Focusable via VO navigation Characters are masked and not announced by VO as entered.
<input type="radio"> input [radio] Radio button Announces as "selected/unselected, radio button" + position of radio button out of total number in radio group VO + space to select Recognizes position within radio group.
<input type="range"> input [range] Range input using a slider Announces as current value + "slider" Right/Left Arrows to adjust value Allows setting a value within a defined range.
<input type="reset"> input [reset] Resets form fields Announces "reset, button" or value content + "button" if value attribute is set VO + space to activate Resets all form inputs to their default values.
<input type="search"> input [search] Search query entry Announces "search text field" Focusable via VO navigation No special interaction.
<input type="submit"> input [submit] Form submission button Announces "submit, button" or value content + "button" if value attribute is set VO + space to activate Submits the form data to the server.
<input type="tel"> input [tel] Telephone number entry Announces as "telephone number field" Focusable via VO navigation Announces "edit text" when input is focused.
<input type="text"> input [text] Single-line text entry Announces as "text field" Focusable via VO navigation Announces "edit text" when input is focused.
<input type="time"> input [time] Time picker control Announces current time value + "time area" VO + Shift + Down Arrow to edit time values, Up/Down Arrow keys to incrementally increase/decrease value Provides an hour/minute input interface.
<input type="url"> input [url] URL entry field Announces "URL field" Focusable via VO navigation Announces "edit text" when input is focused.
<input type="week"> input [week] Week number input Announces "week and year field" Focusable via VO navigation Announces "edit text" when input is focused.
<ins> ins Recently added text Announces "insertion" before inserted text when included as part of a tag with plain text, otherwise no aural indication No special interaction Uses a different tone to indicate deletion. Support was inconsistent across browsers and "insertion" is only ever announced when part of a tag with plain text.
<kbd> kbd User keyboard input representation Reads content as plain text No special interaction No special semantics beyond reading the text.
<label> label Label for input Reads associated label content No special interaction Reads label when associated input gains focus.
<li> li List item Announces as "bullet" if unordered and item number if ordered + list position out of total number of list entries VO + Right Arrow to navigate Announces item count and position.
<main> main Main content Announces "main" VO + Command + Option + Right Arrow Recognized as a main landmark region.
<nav> nav Navigation links Announces as "navigation" VO + Command + Option + Right Arrow Identified as a navigation landmark.
<ol> ol Ordered list Announces as "list" VO + Command + Option + Right Arrow Announces list structure and number of items.
<table> table Data table Announces caption, if present, + "table, X rows and Y columns" VO navigation commands Recognizes row/column headers and cell content.
<ul> ul Unordered list Announces "list with X items" VO + Command + Option + Right Arrow Recognizes the list structure.
