This file contains various tests that can be used for ari-multiline functionality verification.

You can use Braille display to verify if a specific edit is treated as a single line or a multiline edit. Single line edits will have their label displayed prior to edit content on the Braille display. Muiltiline edits will only have content of a current line.

You can also use Script utility mode for verification and make sure that both single line and multiline edits have type code = edit, but subtype code is different: 3 for single line edit and 66 for a multiline.
